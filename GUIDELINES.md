# 초성 낱말판 작업 지침

## 핵심 방향

- 대상은 초등학교 3학년 학생이다.
- 학생 화면은 설명을 최소화하고, 보고 바로 누를 수 있어야 한다.
- 교사 화면은 수업 진행과 관리가 빠르게 가능해야 한다.
- UI는 흰 캔버스, 검은 글자, 파스텔 포인트, 넓은 여백을 기본으로 한다.
- 장식보다 가독성, 터치 편의성, 카드 간 구분을 우선한다.

## 학생 화면 UX

- 최초 화면은 번호 선택만 보여준다.
- 학생 번호는 카드 안 라벨과 같은 디자인의 배지로 좌측 상단에 표시한다.
- 진행률, 참여 인원 같은 보조 수치는 학생 화면에서 숨긴다.
- 주제는 문장형으로 보여준다.
  - 예: `오늘의 주제는 동물 입니다.`
  - 작은따옴표로 감싸지 않는다.
  - 주제 단어 자체를 굵기, 기울기, 밑줄 등 폰트 스타일로 강조한다.
- 별도 입력 섹션을 만들지 않는다.
- 학생은 초성 카드를 눌러 카드 안에서 바로 입력한다.
- 제출 전에는 반드시 한 번 더 확인한다.
  - 입력
  - 확인 화면
  - 최종 저장
- 제출 후에도 본인 낱말은 수정할 수 있다.
- 다른 학생이 채운 칸은 수정할 수 없다.
- 본인 낱말 카드에는 수정 가능함을 작은 연필 아이콘으로 암시한다.

## 교사 화면 UX

- 교사 화면도 같은 주제 문장 형식을 사용한다.
- 주제 설정은 상단의 명확한 컨트롤 영역으로 제공한다.
- 초성 보드는 전체 폭을 충분히 사용한다.
- 참여 현황 섹션은 사용하지 않는다.
- MVP 기능은 사용하지 않는다.
- 교사는 낱말 삭제만 할 수 있다.
- 삭제 버튼은 낱말/학생 번호 배지와 겹치지 않게 카드 우측 상단에 배치한다.
- 날짜별 기록은 하단 관리 영역에 단독으로 넓게 보여준다.

## 초성 카드 디자인

- 카드는 크고 여백이 충분해야 한다.
- 빈 카드는 점선 테두리와 `+` 버튼으로 표시한다.
- 채워진 카드는 초성, 낱말, 학생 번호 라벨을 명확히 보여준다.
- 학생 번호 라벨은 학생별 파스텔 색상을 유지한다.
- 선택된 카드는 일반 카드보다 더 뚜렷하게 보여야 한다.
  - 배경색 변경
  - 테두리 강조
  - 부드러운 그림자
- 선택된 카드가 입력 UI를 포함할 때는 글자가 잘리지 않도록 충분한 폭과 높이를 확보한다.
- 카드 안 액션 버튼은 글자 버튼보다 아이콘 버튼을 우선한다.

## 입력 정책

- 입력은 1~6글자만 허용한다.
- 자음/모음만 입력은 금지한다.
- 빈 문자열, 공백만 입력, 숫자만 입력, 특수문자만 입력은 금지한다.
- 욕설과 무의미한 반복 문자열은 금지한다.
- 선택한 초성과 입력 낱말 첫 글자의 초성이 반드시 일치해야 한다.
- 1글자 완성형 한글 낱말은 허용한다.
  - 예: 개, 소, 말
- 7글자 이상은 막고 `6글자까지 쓸 수 있어요.`라고 안내한다.

## 디자인 원칙

- 작은 표, 작은 배지, 보조 마크를 남발하지 않는다.
- 불필요한 숫자 표시는 제거한다.
- 작은따옴표 같은 텍스트 장식보다 폰트 스타일로 강조한다.
- 버튼은 pill 또는 원형 아이콘 버튼을 사용한다.
- 카드와 작업판은 충분한 내부 여백을 가진다.
- 화면에 남는 공간은 배경 여백이 아니라 작업판 여백으로 보이게 구성한다.
- 학생 화면은 한눈에 읽히는 큰 카드 중심으로 구성한다.
- 교사 화면은 보드와 관리 영역의 위계를 명확히 나눈다.

## 기능 유지/제거 기준

- 유지
  - 학생 번호 선택
  - 교사 번호 0 로그인
  - 주제 설정
  - 랜덤 주제
  - 학생 제출
  - 학생 수정
  - 교사 삭제
  - 날짜별 기록
  - Supabase Realtime

- 제거 또는 사용하지 않음
  - 학생 화면 진행률 표시
  - 학생 화면 참여 인원 표시
  - 교사용 참여 현황 섹션
  - MVP 지정 기능
  - MVP 별 표시

## 구현 주의사항

- 학생 화면에서 별도 제출 패널을 다시 만들지 않는다.
- 카드 내부 입력 UI가 글자를 자르지 않도록 최소 폭과 높이를 확인한다.
- 교사용 삭제 버튼은 학생 번호 라벨과 겹치지 않게 둔다.
- MVP 관련 UI와 API는 다시 연결하지 않는다.
- DB에 남아 있는 `is_mvp` 컬럼은 호환용으로만 보고 앱에서는 사용하지 않는다.
- Supabase가 설정되지 않은 로컬 개발 환경에서도 localStorage 기반 테스트가 가능해야 한다.
- 변경 후에는 항상 다음을 확인한다.

```bash
npm run build
npm run lint
```
